# CLAUDE.md

tuuuui は [Textual](https://textual.textualize.io/) 製のターミナル IDE です。
左にファイラー、中央に Git ビュー / Emacs エディタ、右は tmux 分割ペインのワーク
スペース（claude code / copilot cli）。詳細は `README.md` と
`docs/superpowers/specs/` を参照。

## ディレクトリ

| パス | 内容 |
|------|------|
| `src/tuuuui/` | アプリ本体（`app.py`、`core/`、`widgets/`、`launcher.py`） |
| `tests/` | テスト一式（pytest + Textual パイロット） |
| `docs/superpowers/specs/` | 設計ドキュメント |

`core/`（git・buffers・tmux）は UI 非依存の純ロジック、`widgets/` は Textual UI、
という責務分離を保つこと。

## 開発フロー

1. 大きな変更は `docs/superpowers/specs/` に設計、必要なら
   `docs/superpowers/plans/` に実装プランを作成し、ユーザー確認を得てから着手。
2. 実装後、**動作確認の前にセルフレビュー**（下記チェックリスト）を挟む。
3. テスト・動作確認後にコミット。
4. push / fetch は **SSH 認証のみ**で行う。トークン埋め込み HTTPS や `gh auth`
   系コマンドは使用しない。SSH が失敗したら、トークンへフォールバックせず
   ユーザーに SSH 設定の確認を依頼する。

## セルフレビュー（動作確認の前に必ず）

動作確認の前にセルフレビューを挟む。観点:

- [ ] **(1) UI / ロジック / インフラの分離** — `widgets/`（UI）と `core/`（ロジック・
      git / tmux 等のインフラ）の責務分離が崩れていないか。
- [ ] **(2) 個人情報・機密情報のハードコードなし** — 下記「レビューガイドライン」参照。
- [ ] **(3) ユーザーデータ保全** — 冪等性・バリデーション。特にファイル保存
      （`C-x C-s`）や git 操作で、ユーザーのデータを破壊・上書きしないこと。
- [ ] **(4) マイグレーション不要の確認** — 永続データ形式に破壊的変更が無いか。

**Why:** CLAUDE.md に定義されたレビュープロセス。

**How to apply:** 実装 → レビューチェックリスト確認 → 動作確認の順で進める。

## レビューガイドライン（必須チェック項目）

コード変更、コミット、PR、レビュー時には**必ず以下を確認**する。

### 機密情報・個人情報の混入チェック

- [ ] **個人を特定し得る情報**（氏名、メールアドレス、電話番号、アカウント名、
      ホームディレクトリの実パス等）がコード／テストデータ／コメント／コミット
      メッセージにハードコードされていないか。
  - テストデータは `"AAA"` `"test-user"` 等の架空の値を使用すること。
  - 定数・マッピング・コメント等いかなる形でも禁止。
- [ ] **APIキー・トークン・パスワード等の認証情報**が含まれていないか。
- [ ] ユーザーのローカル個人ファイルや実データ（実 DB ファイル等）がコミット対象に
      含まれていないか（`.gitignore` で除外されているか確認）。
- [ ] エラーメッセージ・ログ出力に機密情報・個人情報が出ていないか。

汎用ロジックで対応できない値は、ユーザー設定ファイルや環境変数で管理する。

### コミットメッセージ

- 機能の説明のみにとどめ、個人情報・認証情報・ローカル実パス等を含めない。
- 例 OK: `feat: add buffer switcher (C-x b)`
- 例 NG: 実トークンや個人のメールアドレス・実ファイルパスを含むメッセージ。

## テスト

テストの実行（`.venv/bin/python -m pytest tests/`、`uv run pytest tests/`、
`uv run pytest tests/*` など）は確認不要で実行してよい。

**Why:** テスト実行は安全な読み取り専用操作であり、毎回確認を求める必要はない。

**How to apply:** pytest コマンドは常に自動実行する。
